# 前台聊天演进计划

## 1. 计划定位

本文档统一收口 `ragAdmin` 前台聊天侧的当前有效计划，覆盖以下主题：

- 独立问答前台与 `/api/app` 产品形态
- 前台聊天会话与知识库选择能力
- 受控 Markdown 渲染
- Tavily 联网搜索接入
- 联网来源独立持久化与前端展示

本文档是当前前台聊天主题的唯一主 plan，替代此前分散的架构设计、实施计划与子专题计划。

## 2. 背景

当前仓库已经具备：

- `rag-admin-server` 的管理端主链路、文档解析、向量写入、单知识库问答与会话持久化
- `rag-admin-web` 后台管理台

仍需持续演进的点包括：

- 将后台治理与普通使用者问答体验彻底分离
- 建立独立前台 `rag-chat-web`
- 统一前台接口域 `/api/app/*`
- 强化前台回答展示、联网增强与来源留痕能力

## 3. 当前产品形态结论

### 3.1 终端结构

- 同仓库保留后台管理端 `rag-admin-web`
- 独立建设问答前台 `rag-chat-web`
- 两个前端共用同一个后端 `rag-admin-server`

### 3.2 前台接口边界

- 前台接口统一收敛到 `/api/app/*`
- 前台会话与后台管理会话严格隔离
- 首页通用会话与知识库内会话上下文分离

### 3.3 前台聊天能力

- 支持简洁聊天页
- 支持模型切换
- 支持联网开关
- 支持知识库选择
- 支持首页通用对话与知识库内对话两种入口

## 4. 当前有效实施主题

### 4.1 独立前台与会话模型

- 在聊天域内补足终端类型与知识库绑定关系
- 前台问答入口统一走 `/api/app/*`
- 不单独建设前台账号体系

### 4.2 受控 Markdown 渲染

- 聊天正文采用受控 Markdown 子集
- 后端统一声明 `text/markdown` 为正文内容类型
- 前端仅对白名单 Markdown 渲染
- 不把主回答全文改造成 JSON 结构化输出

### 4.3 Tavily 联网接入

- 维持 `AppChatService -> WebSearchProvider` 调用边界
- 在 `infra.search` 内实现 Tavily Provider
- 联网失败统一降级为空结果，不中断主问答链路
- 日志与健康检查补齐 Tavily 状态

### 4.4 联网来源独立持久化与展示

- 联网来源与知识库引用分开建模
- 历史消息可回放联网来源
- 前端消息卡片下方展示“联网来源”折叠区
- 展示字段固定为标题、网址、发布时间、摘要

## 5. 当前实施范围

### 5.1 后端

- `/api/app` 认证、模型、知识库、聊天相关接口
- 会话终端隔离与知识库关联关系
- Markdown 内容类型透传
- Tavily Provider 与配置装配
- `chat_web_search_source` 持久化与查询回填

### 5.2 前端

- `rag-chat-web` 独立工程与页面框架
- 模型切换、联网开关、知识库选择
- 白名单 Markdown 渲染
- 联网来源折叠展示

## 6. 验收标准

- 独立问答前台可独立运行
- 前台统一走 `/api/app/*`
- 前台会话与后台会话不串上下文
- 聊天正文以受控 Markdown 正常显示
- 开启联网时可获得真实 Tavily 结果并正常降级
- 回答历史可回放联网来源，且与知识库引用分开展示
