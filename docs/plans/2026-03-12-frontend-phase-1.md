# 前端一期阶段计划与执行记录

## 1. 文档目的

本文档用于汇总 2026-03-12 当天前端一期阶段的设计决策与执行记录，统一替代当天按小功能拆分产生的多个细粒度 plan 文档。

本阶段文档覆盖的能力范围包括：

- 知识库创建、编辑、详情
- 知识库文档列表、上传入口、解析触发
- 任务监控、任务重试、任务详情
- 任务与文档的保守联动
- `Vite` 前端拆包优化

后续同类小功能不再单独新增 plan 文档，默认优先追加到阶段文档中。

## 2. 阶段目标

本阶段目标是把管理端前端从“能登录、能看列表”推进到“能围绕知识库、文档、任务形成基础操作闭环”。

本阶段完成后，应至少具备以下能力：

- 知识库的创建、编辑、详情浏览
- 文档接入知识库并触发解析
- 基础任务可观察与失败任务重试
- 任务、知识库、文档三条线开始建立关联

## 3. 范围与边界

### 3.1 本阶段包含

- 登录与基础后台布局
- 知识库列表页
- 知识库创建页
- 知识库编辑页
- 知识库详情页
- 知识库文档列表
- 单文件上传接入知识库
- 手动触发文档解析
- 任务监控页
- 失败任务重试
- 任务详情页
- 任务详情中的保守业务关联信息展示
- `Vite` 基础拆包优化

### 3.2 本阶段不包含

- 文档详情页
- 文档切片浏览
- 文档版本激活与新增版本
- 任务时间线
- 任务详情中的强业务跳转闭环
- 更激进的前端包体优化

## 4. 关键设计决策

### 4.1 路由结构

当前前端采用独立页面路由，而不是弹窗堆叠式工作流：

- `/knowledge-bases`
- `/knowledge-bases/create`
- `/knowledge-bases/:id`
- `/knowledge-bases/:id/edit`
- `/tasks`
- `/tasks/:id`

该决策的原因是：

- 页面边界清晰
- 后续扩展详情、编辑、版本、切片、任务联动更自然
- 不会让列表页或详情页因为功能累加迅速失控

### 4.2 模型选择策略

知识库创建与编辑中的聊天模型、Embedding 模型采用“双模式兼容”：

- 优先读取真实模型列表
- 接口失败时退化为平台默认模型兜底

设计目标是保证知识库主流程不被模型管理接口拖死。

### 4.3 文档接入策略

文档接入采用三段式闭环：

1. 获取上传凭证
2. 预签名地址直传对象存储
3. 新增文档记录

此处前端不使用统一业务 `http` 客户端上传文件，而是用独立 `fetch` 完成对象存储直传，避免鉴权和统一响应解包干扰预签名上传。

### 4.4 文档解析触发策略

文档解析触发放在知识库详情页文档列表中，以行级按钮发起：

- 仅 `PENDING` / `FAILED` 允许触发
- 只锁当前行，不阻塞整个表格
- 成功或失败后刷新文档列表，以后端真实状态为准

### 4.5 任务监控与治理策略

任务页先做最小可观察性，再逐步引入治理动作：

- 先有列表、筛选、分页、刷新
- 再补失败任务重试
- 再补任务详情

这样可以保证每一轮都有真实用户价值，而不是一次性堆出大而空的“任务中心”。

### 4.6 任务联动策略

任务详情中的业务关联遵循“能确定再展示”的原则：

- 当前能明确识别 `DOCUMENT_PARSE` 类型时，保守展示其关联文档 ID
- 不在前端强行推断知识库或其他对象

该决策是为了避免后端字段尚未稳定时出现错误跳转。

### 4.7 包体优化策略

`Vite` 构建采取“最小风险拆包”而不是激进重构：

- `framework`：`vue`、`vue-router`、`pinia`
- `ui`：`element-plus`、`@element-plus/icons-vue`
- `http`：`axios`

同时将 `chunkSizeWarningLimit` 调整为与当前依赖规模匹配的阈值，目标是先消除构建告警并保持拆包稳定。

## 5. 各能力实现要点

### 5.1 知识库

- 创建页与编辑页复用同一表单组件
- 列表页支持创建回跳刷新与编辑回跳刷新
- 详情页作为知识库工作台承载文档列表

### 5.2 文档

- 当前只做文档接入与解析触发闭环
- 暂未进入文档详情、切片浏览、版本管理
- 文档类型通过文件扩展名做前端保守映射

### 5.3 任务

- 任务监控页提供基础筛选与状态颜色区分
- 失败任务支持重试确认与行级 loading
- 任务详情页完整展示错误信息和基础字段

## 6. 验证基线

本阶段代码变更默认至少满足：

- 页面路由可访问
- 关键交互链可跑通
- `npm run build` 通过
- 若涉及构建优化，应明确检查告警是否消失或收敛

## 7. 后续优先级建议

在本阶段基础上，后续前端优先建议如下：

1. 文档详情页
2. 文档版本列表与激活操作增强
3. 文档切片浏览
4. 任务状态流转时间线
5. 任务与文档 / 知识库详情强联动

## 8. 文档收敛规则

从本文档开始，前端一期内的常规小功能不再单独创建 plan 文档。

默认做法：

- 当前阶段内的小功能设计，追加到本文档对应章节
- 只有跨模块、高风险、长周期专题，才允许单独开新文档
- 若某阶段文档过长，再按“前端二期”“后端一期”这类阶段重新拆分
