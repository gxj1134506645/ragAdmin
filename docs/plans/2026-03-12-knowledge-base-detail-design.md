# 知识库详情页设计说明

## 1. 目标

本次实现目标是在知识库创建页、编辑页闭环基础上，补齐“知识库详情页 + 文档列表浏览”最小可用链路：

- 从知识库列表进入详情页
- 详情页展示知识库基础信息与关键配置
- 详情页下方展示该知识库的文档分页列表
- 详情页提供进入编辑页的自然入口

本轮重点是把“知识库 -> 文档”主链第一次真正连起来，而不是只做一个纯只读占位页。

## 2. 方案选择

本次采用“独立详情页 + 同页文档列表”方案。

选择原因如下：

- 和现有创建页、编辑页的独立路由风格保持一致
- 用户先看知识库配置，再看该知识库下的文档情况，信息组织更自然
- 后续扩展文档上传、解析触发、文档详情等能力时，详情页可以直接演进为知识库工作台

路由设计如下：

- 列表页：`/knowledge-bases`
- 详情页：`/knowledge-bases/:id`
- 编辑页：`/knowledge-bases/:id/edit`

## 3. 页面结构

详情页分为上下两部分：

### 3.1 顶部详情卡片

展示知识库核心字段：

- `kbCode`
- `kbName`
- `description`
- `status`
- `chatModel`
- `embeddingModel`
- `retrieveTopK`
- `rerankEnabled`

顶部提供以下操作：

- `编辑知识库`
- `刷新详情`
- 返回列表入口

### 3.2 文档列表区

展示该知识库下文档的分页列表，首版字段建议如下：

- `documentId`
- `docName`
- `docType`
- `parseStatus`
- `enabled`
- `createdAt`

文档区提供独立刷新入口，但本轮不引入上传、触发解析、筛选和批量操作。

## 4. 数据流设计

### 4.1 详情数据流

页面进入后首先调用 `GET /api/admin/knowledge-bases/{kbId}` 获取知识库详情。

处理规则如下：

- 详情成功：渲染顶部信息卡片，并继续加载文档列表
- 详情失败：页面进入整体错误态，提供“返回列表”和“重新加载”

之所以把详情失败定义为页面级失败，是因为没有知识库主体信息时，不应继续展示从属于该知识库的文档数据。

### 4.2 文档列表数据流

在详情成功后调用 `GET /api/admin/knowledge-bases/{kbId}/documents`。

文档列表区独立维护：

- `loading`
- 空态
- 错误态
- 分页状态

这样可以保证：

- 文档列表失败时，顶部知识库详情仍然可用
- 用户可以单独重试文档列表，不影响详情卡片

## 5. 错误处理

错误处理规则如下：

- 详情加载失败：页面级错误态
- 文档列表失败：区块级错误态，不影响详情展示
- 刷新按钮在请求中进入 loading，避免重复操作
- 空态文案明确区分“暂无文档”和“加载失败”

## 6. 本轮边界

本轮只做浏览链路，不引入写操作。

本轮包含：

- 列表页详情入口
- 详情页基础信息展示
- 文档分页列表展示
- 详情页跳转编辑页

本轮不包含：

- 文档上传
- 触发解析
- 文档详情页
- 文档版本管理
- 文档筛选与批量操作

## 7. 验证范围

本轮至少完成以下验证：

- 列表页可以进入详情页
- 详情页可成功加载知识库基础信息
- 文档列表支持分页与刷新
- 文档列表失败时不影响顶部知识库详情展示
- 前端项目可通过 `npm run build`

## 8. 后续建议

在详情页完成后，建议继续按以下顺序推进：

1. 文档上传入口
2. 触发文档解析
3. 文档详情与切片浏览
